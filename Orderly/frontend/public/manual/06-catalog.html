6. Product Catalog

Browsing the Catalog

placeholder

The Product Catalog displays all of your products in a sortable table. Each row shows the product's ID, Name, Category, Supplier, and Description. Click any column header to sort the list by that column; click it again to reverse the order.

The search bar at the top of the page filters the list as you type, matching against product name, category, supplier, and description. Click Clear Filters to remove the search and return to the full catalog.

Each product row has three action buttons. Edit opens the product form so you can update its details. Delete permanently removes the product. Edit Options expands the row in place to show the product's variants — each with its name, SKU, unit price, stock quantity, and reorder level — and gives you a form to add or modify them without leaving the catalog view.

The toolbar also provides quick access to Export and Print for the full catalog, as well as buttons to create a new product or add a new supplier.

Adding a New Product

placeholder

To add a new product, click Create New Product in the catalog toolbar. This opens the product creation form.

Product Name and Category are required. The category must already exist — if you have not yet created the relevant category, set it up in the Categories section first and then return here. Supplier and Description are optional but recommended for keeping your catalog organized.

The Options checkboxes let you indicate whether the product has variants and whether it supports modifier groups. Has Variants is ticked by default, as most products will have at least one variant (for example, a size or a default option). Has Modifiers should be ticked if customers will be able to make add-on selections when ordering this product, such as extras or customizations. These settings determine what will be available to configure after the product is created.

When creating a new product, the form also includes fields to set up the product's first variant straight away: a Variant Name (such as Small, 12oz, or Default — the product name is used if left blank), a SKU, a Unit Price, and a Stock Quantity. Filling these in saves you a separate step after saving. Additional variants can be added later from the catalog view.

You can optionally upload a Product Image using the file picker at the bottom of the form. A preview is shown once a file is selected. Click Create Product to save, or Cancel to return to the catalog without saving.

Editing a Product

placeholder

To edit an existing product, click the Edit button in its row on the catalog page. This opens the same form used to create a product, pre-filled with the product's current details.

You can update the product's name, category, supplier, description, image, and its Has Variants and Has Modifiers settings. When editing, the first-variant fields shown during creation are not present — variants are managed separately using the Edit Options panel directly in the catalog view.

Click Update Product to save your changes, or Cancel to discard them and return to the catalog.

Variants and SKUs

placeholder

Variants represent the individual purchasable options of a product — for example, sizes, volumes, or styles. Each variant has its own SKU, unit price, stock quantity, and reorder level, and is tracked independently in your inventory.

To manage a product's variants, click Edit Options in the product's row. This expands a panel below the row showing all existing variants in a table, along with a form to add a new one. Fill in the Variant Name, SKU, Unit Price, and optionally the Stock Quantity and Reorder Level, then click Add Option to save it.

To edit an existing variant, click Edit in its row within the expanded panel. The form above the table will populate with that variant's current values — make your changes and click Save Option. To remove a variant entirely, click Delete in its row. Deleted variants cannot be recovered.

Every product should have at least one variant. If a product has only one option with no meaningful name — for example, a standard item sold at a single price — use the product's name or a label like Default as the variant name.

Modifier Groups and Options

placeholder

Modifier groups allow customers to customise their order when selecting a product — for example, choosing a milk type for a coffee, or selecting add-ons for a meal. Each modifier group belongs to a specific variant and contains a set of options the customer can choose from.

When creating a modifier group, you give it a name (such as Milk Type or Extras), set whether it is required or optional, and define the minimum and maximum number of selections a customer must make. Each option within the group has a name and an optional price adjustment that is added to the variant's base price when selected.

Modifier groups only appear on the storefront for products that have Has Modifiers enabled. If a product does not require customisation, leave this setting unticked and no modifier prompts will be shown to customers.

Deleting a Product

placeholder

To delete a product, click the Delete button in its row on the catalog page. A confirmation prompt will appear asking you to confirm the action. Click OK to proceed or Cancel to go back.

Deleting a product permanently removes it and all of its associated variants from your catalog. This action cannot be undone. Orders that already reference the product are not affected — their records remain intact.
