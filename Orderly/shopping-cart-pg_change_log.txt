Shopping Cart Page Change Log
====================
Branch: feat-shopping-cart-pg
Date: 2026-03-31

Files Changed and Changes Made
-------------

1. frontend/src/components/ProductCard.jsx
   - Fixed wrong localStorage key when reading the auth token.
     Was: localStorage.getItem("access")
     Now: localStorage.getItem("accessToken")
   - This caused all Add to Cart requests to be sent without
     authorization, resulting in 401 errors from the API.

2. frontend/src/pages/Cart.jsx
   - Fixed same wrong localStorage key (same root cause as above).
   - Fixed cart never showing items: the draft order endpoint
     (POST /api/v1/orders/draft) only returns an order ID, not
     the item list. Added a second request to GET /api/v1/orders/{id}
     to retrieve the full order detail including items.
   - Fixed incorrect field name when displaying item price.
     Was: item.totalPrice (does not exist on the response)
     Now: item.itemTotal (matches the API response schema)
   - Updated cart item display order: product name shown first,
     variant name beneath it, then modifiers listed individually,
     then item price.
   - Fixed modifier price field name: was m.priceAdjustment,
     now m.priceAdjustmentCharged (matches API response schema).
   - Added increment (+), decrement (−), and delete buttons per
     line item (all use PATCH /api/v1/orders/items/{id}).
     Decrementing to 0 removes the item (same as delete).
   - Added Edit button per line item — navigates to the product
     detail page for that item (requires productId, see below).

3. backend/orders/serializers.py (DraftOrderItemSerializer)
   - Added productId field (source: variant.product.id) so the
     cart Edit button can navigate to the correct product page.

4. frontend/src/pages/ProductPage.jsx
   - Fixed same wrong localStorage key when reading the auth token.
     Was: localStorage.getItem("access")
     Now: localStorage.getItem("accessToken")
   - This caused Add to Cart on the product detail page to fail
     with 401 errors (same root cause as ProductCard.jsx).

5. frontend/src/pages/Login.jsx
   - Removed duplicate calls to setLoggedIn(true) and navigate("/")
     that were firing twice on successful login.

6. backend/orders/services.py (add_item_to_order)
   - Removed merge behavior from B3.2.2 that combined duplicate
     variants into a single row by incrementing quantity.
   - Now always creates a new OrderItem per Add to Cart action
     so each line item can be customized and edited independently.

7. frontend/src/pages/ProductPage.jsx (addModifiers)
   - Modifier POST requests were missing the required quantity field,
     causing the backend to reject all modifiers silently.
   - Added quantity: 1 to the request body so selected options
     (bread type, toppings, etc.) are correctly saved to the cart item.

8. frontend/src/pages/Cart.jsx + frontend/src/pages/ProductPage.jsx (edit cart item)
   - Edit button now passes editItem (itemId), variantId, and modifiers
     (comma-separated optionIds) as URL query params to the product page.
   - Product page reads those params to pre-select the correct variant
     and modifier options when opened in edit mode.
   - In edit mode, the old order item is deleted (PATCH quantity=0) before
     the updated item is added, preventing duplication.
   - Button label changes to "Update Item" in edit mode.

9. frontend/src/pages/Cart.jsx (additional features)
   - Added "Order for [First Last]" heading pulled from localStorage user object.
   - Added subtotal, tax, and grand total lines at the bottom of the cart.
     Subtotal is calculated client-side by summing item totals; tax and
     grand total come from the order detail response.
   - Wrapped entire page in <div className="cart-pg"> for CSS targeting.

10. backend/orders/services.py (recalculate_order_totals)
    - Tax was hardcoded to $0.00 (placeholder).
    - Now calculates at Wake County, NC rate: 7.25%
      (4.75% NC state + 2.5% Wake County/transit), rounded to cents.
