# Full Test Suite for Authentication + Password Reset

This document lists all backend, frontend, and integration tests required to verify the full authentication and password reset system. I conducted all of these tests independently before submitting the code.

---

# 🐍 Backend Tests (Django)

## 1. User Registration
- POST `/api/v1/auth/register/` with valid data → 201
- POST with missing fields → 400
- POST with duplicate email → 400
- User is created in DB
- Password is hashed (not plain text)

---

## 2. Login
- POST `/api/v1/auth/login/` with valid credentials → 200 + session cookie
- POST with wrong password → 400
- POST with non‑existent email → 400
- Session cookie present in response

---

## 3. Logout
- POST `/api/v1/auth/logout/` → clears session cookie
- Logout when not logged in → still returns 200 (idempotent)

---

## 4. Authenticated User Endpoint
- GET `/api/v1/auth/me/` when logged in → returns user data
- GET `/api/v1/auth/me/` when not logged in → 403

---

## 5. Password Reset Request
- POST `/api/v1/auth/password-reset/` with valid email → 200
- POST with non‑existent email → still 200 (no user enumeration)
- Email stored in `mail.outbox`
- Email contains valid UID + token
- Debug link prints correct URL

---

## 6. Password Reset Confirm
- POST `/api/v1/auth/password-reset/confirm/` with valid uid + token + new password → 200
- Invalid token → 400
- Expired token → 400
- Mismatched passwords (if applicable) → 400
- Password updated in DB
- Old password no longer works
- New password works

---

## 7. Token Validity
- Token invalid after password change
- Token invalid after login (Django default)
- Token invalid after timeout (`PASSWORD_RESET_TIMEOUT`)

---

# ⚛️ Frontend Tests (React)

## 8. Registration Page
- Validates required fields
- Shows error messages
- Successful registration shows success state or redirects

---

## 9. Login Page
- Validates email + password
- Shows error on invalid credentials
- Successful login redirects to dashboard
- Cookie is set

---

## 10. Request Password Reset Page
- Validates email
- Shows success message regardless of email existence
- Calls correct API endpoint

---

## 11. Reset Password Page
- Extracts `uid` and `token` from URL
- Shows “invalid link” if missing
- Validates password + confirmation
- Sends correct payload to backend
- Shows success message or redirects

---

## 12. Navigation / Routing
- Protected pages redirect to login when not authenticated
- Authenticated users can access dashboard
- Logout clears session and redirects

---

# 🔗 Integration Tests (End‑to‑End)

## 13. Full Password Reset Flow
1. User requests password reset  
2. Backend generates UID + token  
3. Debug link prints correct URL  
4. User opens reset page  
5. User enters new password  
6. Backend validates token  
7. Password updates  
8. User logs in with new password  
9. Old password fails  

---

## 14. Full Auth Flow
