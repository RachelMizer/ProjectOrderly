# Privacy Policy

Last updated: April 26, 2026

## 1. Overview

Orderly is a full-stack ordering and business management platform created for academic and demonstration purposes. The application allows customers to create accounts, browse products, customize orders, submit orders, and manage basic profile information. It also allows business users and administrators to manage products, inventory, sales information, and order-related data.

This Privacy Policy explains what information Orderly may collect, how that information is used, how it is stored, and how users can request changes or deletion of their information.

## 2. Information We Collect

Orderly may collect the following types of information when users interact with the application:

### Account Information

- Name or username
- Email address
- Password credentials, stored using secure password hashing
- User role, such as customer or business/admin user
- Email verification or password reset status

### Customer and Profile Information

- Customer profile details provided by the user
- Contact or account-related information needed to manage the user account

### Order and Shopping Information

- Products viewed or selected
- Product variants and modifiers selected by the user
- Shopping cart information
- Draft, pending, completed, or canceled order records
- Order history and receipt-related information

### Business and Admin Information

- Product records
- Product categories, variants, and modifiers
- Inventory and stock level information
- Sales summary data
- Supplier or business-related records, if enabled in the application

### Technical Information

- Login and authentication activity
- API requests needed for application functionality
- Error logs or debugging information used to maintain the application

## 3. How We Use Information

Orderly uses collected information to:

- Create and manage user accounts
- Authenticate users and protect account access
- Separate customer and business/admin permissions
- Allow customers to browse products and submit orders
- Maintain carts, order history, and receipts
- Allow business users to manage products, inventory, and sales data
- Support testing, debugging, and application maintenance
- Improve application reliability and security

Orderly does not sell user information.

## 4. Authentication and Access Control

Orderly uses authentication and role-based access control to protect user data. Customer users and business/admin users have different permissions. Protected API endpoints require valid authentication, and admin-level features are restricted to authorized business users.

Authentication tokens should be handled securely and should not be shared. Users are responsible for keeping their login credentials private.

## 5. Data Storage

Orderly stores application data in a database used by the backend system. This may include user accounts, customer profiles, business profiles, products, inventory records, carts, and orders.

For local development and testing, seed data may be used to create sample users, products, and orders. Seed data is intended for development and demonstration purposes only and should not contain real personal information.

## 6. Data Security

Orderly is designed to use reasonable security practices, including:

- Password hashing
- Token-based authentication
- Server-side role checks
- Input validation
- Restricted access to protected endpoints
- Separation of customer and business/admin permissions

No system can guarantee complete security. Users and developers should avoid storing sensitive real-world personal, payment, or financial information in development or demo environments.

## 7. Payment Information

Payment gateway integration is listed as a future or stretch goal. Unless a payment provider is added, Orderly should not collect or store payment card numbers, bank account numbers, or other financial payment credentials.

If payment processing is added in the future, payment information should be handled by a trusted third-party payment provider instead of being stored directly by Orderly.

## 8. Sharing of Information

Orderly does not sell personal information.

Information may be shared only when necessary to:

- Run the application
- Provide account, order, or business management features
- Debug or maintain the system
- Comply with legal, academic, or security requirements
- Support hosting, database, or infrastructure services used by the project

## 9. Data Retention

Orderly keeps user, order, product, and business records for as long as needed to support application functionality, testing, documentation, and academic review.

Users may request that their account or related personal information be updated or removed, subject to project, academic, technical, or legal limitations.

## 10. User Choices and Requests

Users may request to:

- Review their account information
- Update inaccurate profile information
- Delete their account information
- Ask questions about how their information is used

Requests should be sent to the project team or repository maintainers.

## 11. Children’s Privacy

Orderly is not intended for use by children under 13. The application should not knowingly collect personal information from children.

## 12. Development and Demo Notice

Orderly is an academic capstone project. The application may be used in local development, testing, staging, or demonstration environments. These environments may not have the same protections as a production system.

Do not enter real sensitive personal information, payment information, or confidential business information into a demo or test version of the application.

## 13. Changes to This Policy

This Privacy Policy may be updated as Orderly changes. Updates should be committed to the repository with the revised “Last updated” date.

## 14. Contact

For questions about this Privacy Policy or the handling of user information, contact the Orderly project team or repository maintainers.
